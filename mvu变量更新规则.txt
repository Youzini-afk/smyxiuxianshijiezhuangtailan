---
变量更新规则
  世界定位
    当前大域, 当前区域, 当前场景
      check
        - 移动到新地点时更新，描述需简洁准确
    场景类型
      type '野外''宗门''城镇''秘境''战场''海域''地府''王朝''商会''酒家''禁地'
      check
        - z.enum 约束，必须使用列出的值
    当前事件
      type '宗门大比''秘境''历练任务''境界突破''炼器''炼丹''奇遇''机缘''宗门战争''拍卖会''阵法''符箓''驭兽''狩猎''宗门职务'''
      check
        - z.enum 约束，触发对应活动时设置，活动结束后清为空字符串
        - 该变量是世界书中动态事件指南的主触发器
    日期
      format 阴阳历X年X季X月X日
      check
        - 每次事件推进、休息、旅行或闭关后合理推进
    天气
      check
        - 场景变化或时间推进时酌情更新

  在场人物
    type -
      {
        [角色名 string] {
          境界 string;
          年龄 number;
          势力 string;
          武器 string;
          功法 string;
        }
      }
    check
      - 角色出场时 insert（包含境界年龄势力武器功法等基础信息），离场时 remove
      - 角色名必须精确（如殷冬雪而非冬雪）
      - 信息来源于角色人设，必须准确填写，不可编造
      - 龙套NPC也要记录基础信息（境界可写凡人等）
      - 角色信息发生变化时（如突破境界、更换武器）及时更新

  用户信息
    姓名, 性别
      check
        - 开局初始化后不再更改
    年龄
      type number
      check
        - 闭关或时间跨度事件后必须增长
    灵根
      check
        - 先天属性，初始化后永不更改
        - 格式：品级+数量+属性，如 天灵根·单金  地灵根·金水双  玄灵根·金木水三  杂灵根·五行俱全
        - 变异灵根示例：天灵根·纯雷  地灵根·纯冰
    体质
      check
        - 多数人无特殊体质，保持空字符串
        - 发现特殊体质后填入（如冰晶玉体天生剑体先天道体等）
        - 一旦确定不可随意更改
    称号
      check
        - 记录用户当前最重要的身份头衔
        - 随身份变化更新（如外门弟子→内门弟子→核心弟子→长老）
    修真百艺
      type string[]
      check
        - 常见百艺：剑修丹修阵法师炼器师符篆师灵植师驯兽师卜算师体修法修
        - 学习新百艺需要明确的学习过程（拜师获得典籍长期修炼），不能因为尝试一次就添加
        - 若包含'剑修'才激活管理剑心境界，若包含'丹修'才激活管理丹道境界
    修为境界
      type '凡人''炼气一层'~'炼气十层''筑基前期'~'筑基大圆满'...'真仙境一重天'~'真仙境大圆满'
      check
        - z.enum 约束，仅在突破成功后更新为下一阶段值
    修炼进度
      type number
      range 0~100
      check
        - 日常修炼 +3~8  闭关 +8~15  生死战斗 +10~20  服用丹药 +5~15  传承顿悟 +15~30
        - 达到 100 时设当前状态为'突破在即'，突破成功后重置为 0
        - 突破失败进度降至 70~80，当前状态变为'受伤'或'重伤'
        - 真仙境大圆满时不可再增长
    所属势力
      check
        - 加入离开被逐出势力时更新
    当前状态
      type '正常''受伤''重伤''濒死''修炼''闭关''战斗''突破在即''突破中''中毒''昏迷''被困''疗伤''赶路'
      check
        - z.enum 约束，根据当前情况选择对应值
    剑心境界
      type '入剑''剑意''剑势''剑心''剑心通明''无我''无剑''无我无剑'
      check
        - z.enum 约束，剑修专属（修真百艺含'剑修'时才管理），非剑修保持空
        - 剑心是灵根的平替：杂灵根+高剑心=修为低但剑道强，天灵根+低剑心=修为高但剑道平庸
        - 提升极慢，需要剑道顿悟或生死战斗
    丹道境界
      type '炼精化气''炼气化神''炼神还虚''炼虚合道'
      check
        - z.enum 约束，丹修专属（修真百艺含'丹修'时才管理），非丹修保持空
        - 丹道是灵根的平替，通过大量炼丹实践和领悟提升
    宗门贡献
      type number
      check
        - 完成宗门任务 +10~50  参与宗门战争 +50~500
        - 兑换法宝、学习功法、进入禁地时扣除

    背包
      灵石.${下品中品上品}
        type number
        check
          - 交易、拾取、消耗灵石时更新
          - 兑换比例：1中品 = 1000下品，1上品 = 1000中品
      丹药
        type -
          {
            [丹药名 string] {
              数量 number;
              品级 '凡丹·下品'...'仙丹·极品';
              描述 string;
            }
          }
        check
          - 品级使用 z.enum 约束
          - 数量 = 0 时 Schema 自动移除该条目
      材料
        type -
          {
            [材料名 string] {
              数量 number;
              品级 string;
            }
          }
        check
          - 数量 = 0 时 Schema 自动移除该条目
      杂物
        type -
          {
            [物品名 string] {
              数量 number;
              描述 string;
            }
          }
        check
          - 数量 = 0 时 Schema 自动移除该条目

    功法
      主修功法
        type -
          {
            名称 string;
            品阶 string;
            总层数 number;
            当前层数 number;
            描述 string;
          }
        check
          - 日常只更新当前层数（突破时 +1）
          - 更换主修功法时，名称品阶总层数当前层数描述全部替换
      辅修功法
        type -
          {
            [功法名 string] {
              品阶 string;
              总层数 number;
              当前层数 number;
              描述 string;
            }
          }
        check
          - 习得新辅修功法时添加（需明确学习过程），含品阶总层数
          - 日常只更新当前层数
      术法
        type -
          {
            [术法名 string] {
              熟练度 '入门''小成''大成''精通''圆满';
              描述 string;
            }
          }
        check
          - 熟练度使用 z.enum 约束，提升需大量实战，不能一次跳多级

    法宝
      ${本命法宝防御法宝}
        type -
          {
            名称 string;
            品阶 '法器''灵器''法宝''灵宝''仙器';
            描述 string;
          }
        check
          - 品阶使用 z.enum 约束
          - 祭炼、装备或更换时更新
      ${辅助法宝未装备法宝}
        type -
          {
            [法宝名 string] {
              品阶 '法器''灵器''法宝''灵宝''仙器';
              描述 string;
            }
          }
        check
          - 品阶使用 z.enum 约束
          - 获得新法宝时添加到未装备，装备时移动到对应位

    灵兽
      type -
        {
          [灵兽名 string] {
            品种 string;
            境界 string;
            描述 string;
          }
        }
      check
        - 驯服收服灵兽时添加，包含品种境界描述
        - 灵兽境界提升时更新
        - 灵兽死亡或放生时 remove

  男性好感度
    type -
      {
        [角色名 string] {
          好感度 number;  range -1000~1000
          关系阶段 '仇敌''生死大仇''厌恶''敌意''陌生''冷淡''素不相识''萍水相逢''点头之交''泛泛之交''相识''相知''知己''莫逆之交''生死之交';
          重要事件 string;
        }
      }
    check
      - 男性角色走兄弟情义路线，绝对禁止出现爱情暧昧道侣
      - 关系上限为生死之交，这是男性友谊的最高境界
      - 好感度区间：[-1000~-500]仇敌生死大仇 [-500~-200]厌恶敌意 [-200~0]陌生冷淡 [0~100]素不相识 [100~200]萍水相逢 [200~300]点头之交 [300~400]泛泛之交 [400~500]相识 [500~600]相知 [600~800]知己 [800~900]莫逆之交 [900~1000]生死之交
      - 每月上限 +50，单次互动上限 +40
      - 首次遇见男性角色时用 insert 添加到男性好感度，好感度初始为 0
      - 重要事件：记录与该角色之间发生的重大转折（简短1~2句话）

  女性好感度
    type -
      {
        [角色名 string] {
          好感度 number;  range -1000~1000
          关系阶段 '仇敌''生死大仇''厌恶''敌意''陌生''冷淡''素不相识''萍水相逢''点头之交''泛泛之交''相识''相知''知己''红颜知己''情愫暗生''两情相悦''道侣''双修道侣''生死相依''不离不弃';
          重要事件 string;
        }
      }
    check
      - 女性角色走友情→爱情路线，红颜知己需要特殊契机才能突破
      - 每月上限 +50，单次互动上限 +40
      - 性格修正系数：极度冷淡 x0.4~0.6  冷淡 x0.6~0.8  中性 x0.8~1.0  温暖 x1.0~1.2
      - 阶段减速：[0~300] 正常  [300~700] x0.8  [700+] x0.6
      - 好感度区间：[-1000~-500]仇敌生死大仇 [-500~-200]厌恶敌意 [-200~0]陌生冷淡 [0~100]素不相识 [100~200]萍水相逢 [200~300]点头之交 [300~400]泛泛之交 [400~500]相识 [500~600]相知 [600~700]知己 [700~800]红颜知己 [800~900]情愫暗生 [900~950]两情相悦 [950~980]道侣 [980~990]双修道侣 [990~995]生死相依 [995~1000]不离不弃
      - 红颜知己 != 爱情，不是所有女角色都会爱上用户，红颜知己本身就是美好的终点
      - 从红颜知己到情愫暗生必须有特殊契机（生死与共长久相处心动瞬间）
      - 首次遇见女性角色时用 insert 添加到女性好感度，好感度初始为 0
      - 重要事件：记录与该角色之间发生的重大转折（简短1~2句话）

  剧情标记
    type -
      {
        [标记名 string] string  简短剧情描述
      }
    check
      - 触发重要剧情节点时添加，值为简短描述（1~2句话）

  开局初始化
    check
      - 若用户信息的姓名性别年龄灵根修为境界等字段为空，必须根据开局剧情初始化所有用户信息字段
      - 灵根格式必须遵循品级+数量+属性（如 天灵根·单金  玄灵根·金木水三）
      - 若用户是剑修，剑心境界初始为'入剑'；若用户是丹修，丹道境界初始为'炼精化气'
      - 非剑修非丹修对应境界保持空字符串
