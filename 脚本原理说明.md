## 核心原理

### 整体架构

```
酒馆助手 (SillyTavern)
    ↓
MVU Beta 框架 (变量管理)
    ↓
状态栏脚本 (读取并显示)
    ↓
前端页面 (可视化展示)
```

### 工作流程

1. **初始化阶段**：等待 MVU 框架加载完成
2. **数据读取阶段**：从 MVU 获取变量数据
3. **数据处理阶段**：解析并提取角色信息
4. **渲染阶段**：生成 HTML 并显示卡片
5. **交互阶段**：监听事件，支持刷新/切换样式等操作

## MVU框架数据读取

### 1. 检测 MVU 是否可用

```javascript
function waitForMvu(callback, maxAttempts = 20) {
    let attempts = 0;
    
    const checkMvu = () => {
        attempts++;
        
        // 检查 MVU 是否存在（支持多种环境）
        const mvuExists = typeof Mvu !== 'undefined' ||
                         typeof window.Mvu !== 'undefined' ||
                         (window.parent && typeof window.parent.Mvu !== 'undefined');
        
        if (mvuExists) {
            // 如果 MVU 在父窗口，引用它
            if (typeof Mvu === 'undefined' && window.parent && typeof window.parent.Mvu !== 'undefined') {
                window.Mvu = window.parent.Mvu;
            }
            callback();
        } else if (attempts < maxAttempts) {
            setTimeout(checkMvu, 1000);
        } else {
            console.error('MVU框架加载超时');
            callback();
        }
    };
    
    checkMvu();
}
```

## 关键函数解析

### 1. 判断是否为角色数据

```javascript
function isCharacterData(obj) {
    if (!obj || typeof obj !== 'object') {
        return false;
    }
    
    // 检查是否包含核心字段
    const coreFields = ['心跳', '基础信息', '服装', '身体特征', '性经验'];
    let matchCount = 0;
    
    for (const field of coreFields) {
        if (obj[field] !== undefined) {
            matchCount++;
        }
    }
    
    // 至少匹配3个核心字段才认为是角色数据
    return matchCount >= 3;
}
```

### 2. 智能查找角色数据

```javascript
function findCharacterData(statData) {
    let bondsData = {};
    let dataSource = '';
    
    // 方式1: 查找"人物羁绊"字段
    if (statData['人物羁绊']) {
        bondsData = statData['人物羁绊'];
        dataSource = '人物羁绊';
    }
    // 方式2: stat_data本身就是角色数据
    else if (isCharacterData(statData)) {
        bondsData = { '当前角色': statData };
        dataSource = 'stat_data直接数据';
    }
    // 方式3: 查找所有可能是角色的子字段
    else {
        const possibleCharacters = {};
        for (const key of Object.keys(statData)) {
            if (key !== '$meta' && typeof statData[key] === 'object' && isCharacterData(statData[key])) {
                possibleCharacters[key] = statData[key];
            }
        }
        
        if (Object.keys(possibleCharacters).length > 0) {
            bondsData = possibleCharacters;
            dataSource = '自动识别角色字段';
        }
    }
    
    return { bondsData, dataSource };
}
```

### 3. 读取具体数值

MVU 数据格式是 `[值, 描述]`，所以读取时需要 `[0]`：

```javascript
// ❌ 错误的读取方式
const name = beauty.基础信息.姓名;  // 得到: ["秋青子", "描述"]

// ✅ 正确的读取方式
const name = beauty.基础信息?.姓名?.[0];  // 得到: "秋青子"
```

#### 安全读取（防止数据不存在）

```javascript
// 使用可选链操作符 ?. 和空值合并 ||
const name = beauty.基础信息?.姓名?.[0] || '未知';
const age = beauty.基础信息?.年龄?.[0] || '-';
const heartbeat = beauty.心跳?.[0] || 80;

// 读取嵌套数据
const height = beauty.基础信息?.外貌?.身高?.[0] || '-';
const bust = beauty.身体特征?.三围?.胸围?.[0] || '-';

// 读取服装（可能为空字符串表示脱掉了）
const top = beauty.服装?.上衣?.[0] || '无';
const shoes = beauty.服装?.鞋子?.[0] || '无';  // 如果是 ""，显示为 "无"
```

#### 读取扩展字段（敏感部位）

```javascript
let sensitiveParts = [];

if (beauty.敏感部位) {
    // 获取所有部位名称
    const parts = Object.keys(beauty.敏感部位).filter(k => k !== '$meta');
    
    parts.forEach(part => {
        const reaction = beauty.敏感部位[part]?.反应描述?.[0] || '-';

            name: part,      // 部位名称
```

## 实用技巧

### 1. 数据不存在时的处理

```javascript
// 使用 || 提供默认值
const value = data?.[0] || '默认值';

// 使用 ?? 只在 null/undefined 时提供默认值（保留空字符串）
const value = data?.[0] ?? '默认值';
```

### 2. 监听数据更新

```javascript
// 监听 MESSAGE_RECEIVED 事件，自动刷新数据
if (typeof eventSource !== 'undefined' && eventSource.on) {
    eventSource.on('MESSAGE_RECEIVED', () => {
        // 延迟一下，等待变量处理完成
        setTimeout(() => {
            loadAndDisplay();
        }, 500);
    });
}
```

### 3. 重新处理变量

```javascript
async function reprocessCurrentMessageVariables() {
    // 获取当前消息ID
    let messageId = getLastMessageId();
    
    // 查找最后一条AI消息
    while (messageId >= 0) {
        const message = getChatMessages(messageId).at(-1);
        if (message && message.role !== 'user') {
            break;
        }
        messageId--;
    }
    
    // 方法1: 直接调用处理函数
    if (typeof window.parent?.handleVariablesInMessage === 'function') {
        await window.parent.handleVariablesInMessage(messageId);
        return;
    }
    
    // 方法2: 清空后重新处理
    if (typeof updateVariablesWith === 'function') {
        await updateVariablesWith(vars => {
            _.unset(vars, 'stat_data');
            _.unset(vars, 'delta_data');
            _.unset(vars, 'display_data');
            _.unset(vars, 'schema');
            return vars;
        }, { type: 'message', message_id: messageId });
        
        if (typeof window.parent?.handleVariablesInMessage === 'function') {
            await window.parent.handleVariablesInMessage(messageId);
        }
    }
}
```

## 总结

### 核心要点

1. **MVU 框架是数据源**：所有变量数据都存储在 MVU 中
2. **数据格式是 `[值, 描述]`**：读取时必须用 `[0]` 获取实际值
3. **支持多种数据结构**：自动识别不同的数据组织方式
4. **使用可选链**：防止数据不存在导致报错
5. **监听事件自动更新**：接收到新消息时自动刷新显示

### 关键API

```javascript
// 获取数据
Mvu.getMvuData({ type: 'message', message_id: 'latest' })

// 获取消息
getLastMessageId()
getChatMessages(messageId)

// 更新变量
updateVariablesWith(callback, { type: 'message', message_id: messageId })

// 重新处理
window.parent?.handleVariablesInMessage(messageId)

// 事件监听
eventSource.on('MESSAGE_RECEIVED', callback)
```

## 如何编写状态栏脚本

这部分教AI如何从零开始编写一个完整的状态栏系统。

### 整体架构设计

一个完整的状态栏系统包含以下几个部分：

```javascript
// 1. 样式定义 (CSS)
const styles = `<style id="character-status-styles">...</style>`;

// 2. 辅助函数
// - waitForMvu() - 等待MVU框架加载
// - safeGet() - 安全读取MVU数据
// - isCharacterData() - 判断是否为角色数据

// 3. 初始化函数
// - initializeCharacterStatus() - 创建容器、注入样式、启动加载

// 4. 数据读取和渲染函数
// - loadAndDisplay() - 读取MVU数据并渲染
// - generateCharacterCard() - 生成单个角色卡片HTML

// 5. 事件处理
// - setupEventDelegation() - 绑定按钮点击事件

// 6. 启动代码
$(document).ready(() => { ... });
```

### 重要提醒

编写状态栏时最容易犯的错误：
1. ❌ 忘记创建DOM容器，直接操作`$('#status-top')`导致失败
2. ❌ 样式注入位置错误，应该在`initializeCharacterStatus()`中注入
3. ❌ 忘记等待MVU加载就读取数据
4. ❌ 读取MVU数据时忘记用`[0]`获取值

### 正确的编写顺序

**Step 0**: 定义样式（但不立即注入）
**Step 1**: 编写辅助函数（waitForMvu、safeGet、isCharacterData）
**Step 2**: 编写初始化函数（创建容器、注入样式）← 最关键
**Step 3**: 编写数据读取函数（loadAndDisplay）
**Step 4**: 编写卡片生成函数（generateCharacterCard）
**Step 5**: 编写事件处理函数（setupEventDelegation）
**Step 6**: 编写启动代码（$(document).ready）

```javascript
function waitForMvu(callback, maxAttempts = 20) {
    let attempts = 0;
    
    const checkMvu = () => {
        attempts++;
        
        // 检查MVU是否存在
        const mvuExists = typeof Mvu !== 'undefined' ||
                         typeof window.Mvu !== 'undefined' ||
                         (window.parent && typeof window.parent.Mvu !== 'undefined');
        
        if (mvuExists) {
            // 如果MVU在父窗口，引用它
            if (typeof Mvu === 'undefined' && window.parent && typeof window.parent.Mvu !== 'undefined') {
                window.Mvu = window.parent.Mvu;
            }
            callback();
        } else if (attempts < maxAttempts) {
            setTimeout(checkMvu, 1000);
        } else {
            console.error('MVU框架加载超时');
            callback(); // 超时也调用，让后续逻辑处理
        }
    };
    
    checkMvu();
}
```

```javascript
// 判断一个对象是否是角色数据
function isCharacterData(obj) {
    if (!obj || typeof obj !== 'object') {
        return false;
    }
    
    // 定义核心字段（根据你的初始变量定义）
    const coreFields = ['姓名', '性别', '当前所想']; // 简略版
    // 或者 const coreFields = ['心跳', '基础信息', '服装']; // 完整版

            matchCount++;
        }
    }

}
```

```javascript
function loadAndDisplay() {
    console.log('开始加载MVU数据...');
    
    // 第1步：检查MVU是否可用
    if (typeof Mvu === 'undefined') {
        console.error('MVU未加载');
        $('#status-top').html('<p>等待MVU框架加载...</p>').removeClass('hidden');
        return;
    }
    
    // 第2步：获取目标消息ID（找最后一条AI消息）
    let targetMessageId = 'latest';
    if (typeof getLastMessageId === 'function' && typeof getChatMessages === 'function') {
        let currentId = getLastMessageId();
        
        while (currentId >= 0) {
            const message = getChatMessages(currentId).at(-1);
            if (message && message.role !== 'user') {
                targetMessageId = currentId;
                break;
            }
            currentId--;
        }
    }
    
    // 第3步：从MVU读取数据
    const data = Mvu.getMvuData({ 
        type: 'message', 
        message_id: targetMessageId 
    });
    
    // 第4步：检查数据是否存在
    if (!data || !data.stat_data) {
        console.warn('MVU数据为空');
        $('#status-top').html('<p>暂无数据</p>').removeClass('hidden');
        return;
    }
    
    const statData = data.stat_data;
    console.log('stat_data字段:', Object.keys(statData));
    
    // 第5步：查找角色数据（支持多种数据结构）
    let bondsData = {};
    
    if (statData['人物羁绊']) {
        // 情况1：有专门的"人物羁绊"字段
        bondsData = statData['人物羁绊'];
        console.log('找到"人物羁绊"字段');
    }
    else if (isCharacterData(statData)) {
        // 情况2：stat_data本身就是角色数据
        bondsData = { '当前角色': statData };
        console.log('stat_data本身就是角色数据');
    }
    else {
        // 情况3：自动识别可能的角色字段
        const possibleCharacters = {};
        for (const key of Object.keys(statData)) {
            if (key !== '$meta' && typeof statData[key] === 'object' && isCharacterData(statData[key])) {
                possibleCharacters[key] = statData[key];
            }
        }
        
        if (Object.keys(possibleCharacters).length > 0) {
            bondsData = possibleCharacters;
            console.log(`自动识别到 ${Object.keys(possibleCharacters).length} 个角色字段`);
        }
    }
    
    // 第6步：检查是否找到角色数据
    if (!bondsData || Object.keys(bondsData).length === 0) {
        console.warn('未找到角色数据');
        $('#status-top').html('<p>未找到角色数据</p>').removeClass('hidden');
        return;
    }
    
    const characters = Object.keys(bondsData).filter(k => k !== '$meta');
    
    if (characters.length === 0) {
        console.log('没有角色数据');
        $('#status-top').html('<p>暂无角色数据</p>').removeClass('hidden');
        return;
    }
    
    console.log(`找到 ${characters.length} 个角色`);
    
    // 第7步：隐藏加载提示
    $('#status-top').addClass('hidden');
    
    // 第8步：生成卡片HTML
    const cardsHtml = characters.map(charKey => {
        const beauty = bondsData[charKey];
        return generateCharacterCard(charKey, beauty);
    }).join('');
    
    // 第9步：渲染到页面
    $('#character-content').html(cardsHtml);
    
    console.log('数据显示完成');
}
```

**关键点**：
1. 使用`$('#status-top')`和`$('#character-content')`操作DOM，前提是初始化函数已经创建了这些元素
2. 使用`.addClass('hidden')`和`.removeClass('hidden')`控制显示/隐藏
3. 支持多种数据结构（人物羁绊字段、直接数据、自动识别）

```javascript
    // 等待MVU并加载数据
    waitForMvu(() => {
        setTimeout(loadAndDisplay, 500);
    });
    
    // 监听消息事件
    if (typeof eventSource !== 'undefined' && eventSource.on) {
        eventSource.on('MESSAGE_RECEIVED', () => {
            setTimeout(loadAndDisplay, 500);
        });
    }
}

// 启动
$(document).ready(() => {
    initializeCharacterStatus();
});
```

### 关键注意事项

编写状态栏时必须注意的要点：

#### 1. 必须先创建容器（最容易犯的错误！）
❌ **错误**：直接写`$('#status-top').html('...')`
✅ **正确**：先在`initializeCharacterStatus()`中创建容器，再操作

```javascript
// 错误示例 - 容器不存在
$(document).ready(() => {
    $('#status-top').html('加载中...'); // 找不到元素！
});

// 正确示例 - 先创建容器
function initializeCharacterStatus() {
    const statusTop = $('<div>', { id: 'status-top' });
    // ... 创建并插入容器
}
```

#### 2. 样式注入位置
❌ **错误**：在脚本开头写`$('head').append(styles)`
✅ **正确**：在`initializeCharacterStatus()`函数中注入

#### 3. MVU数据格式
MVU数据是 `[值, 描述]` 格式：
```javascript
// 数据格式
"姓名": ["张三", "角色的真实姓名"]

// 读取方式
const name = charData.姓名[0];  // "张三"
const name = safeGet(charData.姓名);  // 更安全
```

#### 4. 函数调用顺序
```javascript
$(document).ready(() => {
    initializeCharacterStatus();  // ✅ 正确：调用初始化
    // loadAndDisplay();  // ❌ 错误：直接调用会失败
});
```

#### 5. 显示/隐藏控制
```javascript
// 隐藏加载提示
$('#status-top').addClass('hidden');

// 显示错误信息
$('#status-top').html('<p>错误</p>').removeClass('hidden');
```

#### 6. 事件监听
在`initializeCharacterStatus()`中注册，实现自动刷新：
```javascript
if (typeof eventSource !== 'undefined' && eventSource.on) {
    eventSource.on('MESSAGE_RECEIVED', () => {
        setTimeout(loadAndDisplay, 500);
    });
}
```

#### 7. 数据结构兼容性
支持多种数据结构：
- `stat_data['人物羁绊']` - 标准结构
- `stat_data` 本身是角色 - 单角色
- 自动识别多个角色字段
